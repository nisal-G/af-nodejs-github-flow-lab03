## Features Implemented

### 1. Console Output
Basic console logging for testing Node.js environment.

### 2. File Reading (`fs.readFile`)
Demonstrates reading content from a text file asynchronously.
- Reads from `file.txt`
- Displays file content in console
- Error handling included

### 3. File Writing (`fs.writeFile`)
Shows how to write data to files using the file system module.
- Writes to `output.txt`
- Success confirmation message
- Error handling included

### 4. Web Server (`http.createServer`)
Creates a basic HTTP server that listens on port 3000 and serves HTML content.
- **Access:** http://localhost:3000
- Serves "Hello World!" message
- HTML formatted response

### 5. HTTP Request (`https.get`)
Makes GET requests to external APIs and processes the response.
- Uses JSONPlaceholder API
- Fetches post data (ID: 1)
- Parses and displays JSON response
- Error handling included

### 6. Custom Modules (`require/module.exports`)
Demonstrates creating and using custom Node.js modules.
- `myModule.myFunction()` - Returns greeting message
- `myModule.add(a, b)` - Performs addition
- `myModule.greet(name)` - Personalized greeting

### 7. Promises
Shows asynchronous operations using JavaScript Promises with resolve/reject.
- Demonstrates promise creation
- `.then()` for success handling
- `.catch()` for error handling
- Timeout simulation

### 8. Async/Await
Modern asynchronous programming using async/await syntax.
- `async function` declaration
- `await` keyword usage
- Try-catch error handling
- Multiple async examples

## GitHub Flow Workflow Used

This project demonstrates the complete GitHub Flow:

1. **Create Branch** - Create feature branch from main
2. **Add Commits** - Make changes and commit with descriptive messages
3. **Open Pull Request** - Push branch and create PR on GitHub
4. **Review & Discuss** - Code review process
5. **Merge** - Merge approved changes to main
6. **Pull Updates** - Synchronize local repository

### Branch Naming Convention

```bash
git status
```

## Troubleshooting

### Port Already in Use
If port 3000 is occupied:
```javascript
// Change port in app.js
.listen(4000); // Use different port
```

### Module Not Found
Ensure you're in the correct directory:
```bash
cd af-nodejs-github-flow-lab03
dir  # Windows - List files (should see app.js, my-module.js)
ls   # Mac/Linux - List files
```

### File Not Found Error
Make sure `file.txt` exists in the project root:
```bash
echo This file will be read by the fs module. > file.txt
```
